# STRATEGIES DIRECTORY (`strategies/`)

This folder contains isolated configurations and execution contexts for disparate trading strategies or exchanges.

## Structure of a Strategy

Each subdirectory (e.g., `fractal_momentum_nse`, `fractal_momentum_nasdaq`) represents a unique instantiation of the core engine. A strategy directory typically contains:

1. `strategy.yaml`: The master configuration file dictating runtime parameters.
2. `universe.csv`: A list of ticker symbols that belong to this strategy/exchange.
3. `.env.<strategy_name>` (Optional): Enforces specific secrets or Database URLs isolated to this runtime.
4. `data/`: A directory dynamically generated by `strategy_loader.py` to hold local caches, `logs/`, and `backtest/` outputs.

## Example `strategy.yaml`

```yaml
name: "Fractal Momentum - Nasdaq"
strategy_type: "fractal_momentum"

exchange:
  suffix: ".US"
  timezone: "America/New_York"
  market_open: "09:30"
  market_close: "16:00"

engine:
  cycle_seconds: 300
  fetch_lookback_days: 10
  compute_lookback_days: 365
  fetch_concurrency: 16
  compute_workers: 8
  order_qty: 1
```

## How they are run
You never run a strategy natively from this folder. Instead, you invoke the global Universal Runner from the repository root:
`python run_strategy.py strategies/fractal_momentum_nasdaq --mode live`

## Rules
- **No Cross-Pollination**: A strategy should never read from or write to another strategy's directory.
- **Universe Integrity**: Make sure the universe specified for a strategy matches its exchange (e.g. `.L` for LSE, `.US` for Nasdaq). Script `rebuild_universe.py` at the root exists specifically to help sanitize these suffixes.
